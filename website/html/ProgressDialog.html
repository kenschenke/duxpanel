DuxProgressDialog

The <DuxProgressDialog> component displays a progress dialog for potentially long operations. A progress bar and abort button are optional. Furthermore, the dialog can delay opening for operations that are sometimes, but not always long such as web service calls.

Property Default Value Description
min 0 The minimum value of the progress bar
max 0 The maximum value of the progress bar
value 0 The current value of the progress bar. It is rendered as a percentage between min and max.
allowAbort false If true, an abort button is rendered in the dialog. The onAbort property is required if true.
onAbort (none) This function is called if the user clicks the abort button.
showAfter 0 For operations which are not always long, but can potentially take a while (such as web service calls), the dialog can delay showing with this property. A timer is started when the show property transitions from false to true. Once the timer elapses, the dialog is displayed if show is still true. This property is the number of milliseconds to delay. If this property is 0, the dialog is displayed immediately when show becomes true.
abortButtonClassName (none) The CSS class(es) to apply to the abort <button>
progressComponent (none) This is a React component to render the progress bar. If not present, no progress bar is rendered. For web sites using Twitter Bootstrap 4, <ProgressBarBS4> is supplied. See below for an example of use. The supplied component is passed the min, max, and value properties.
